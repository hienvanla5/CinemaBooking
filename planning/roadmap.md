# 🗺️ Project Roadmap

This roadmap outlines the overall development plan for the **Cinema Booking System**. The project is scheduled over **6 weeks**, with each week focusing on a specific milestone to gradually build a complete, maintainable, and well-tested console application.

**Project Duration:** 22/06/2026 – 02/08/2026

---

# 🎯 Project Objectives

By the end of the project, the application will demonstrate:

* Object-Oriented Programming (OOP)
* Generic Programming
* CSV File Persistence
* Input Validation
* Custom Exception Handling
* Multi-threading & Thread Safety
* Design Patterns
* Unit & Integration Testing
* Clean Code Principles
* Comprehensive Documentation

---

# 📅 Development Roadmap

| Week       | Date              | Theme                         | Key Technologies                                 | Deliverables                                                                                                                       |
| ---------- | ----------------- | ----------------------------- | ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| **Week 1** | **22/06 – 28/06** | Foundation & Data Layer       | Java, File I/O, Generic Repository, JUnit        | Set up project structure, implement domain models, CSV persistence, Generic Repository, and basic CRUD operations.                 |
| **Week 2** | **29/06 – 05/07** | Business Logic & Validation   | Custom Exceptions, Validator, TDD                | Develop the ticket booking workflow, implement input validation, custom exceptions, and comprehensive unit tests.                  |
| **Week 3** | **06/07 – 12/07** | Theater & Showtime Management | Entity Relationships, Repository Pattern         | Add Theater, Seat, and Showtime management, enhance CRUD operations, and support booking based on showtimes.                       |
| **Week 4** | **13/07 – 19/07** | Concurrent Ticket Booking     | synchronized, ExecutorService, Java Concurrency  | Implement thread-safe booking, simulate concurrent users, prevent race conditions, and evaluate system performance.                |
| **Week 5** | **20/07 – 26/07** | Refactoring & Design Patterns | Singleton, Factory, Strategy, Generic Repository | Refactor the codebase, improve maintainability, apply design patterns, and optimize project architecture.                          |
| **Week 6** | **27/07 – 02/08** | Finalization & Documentation  | Logging, Integration Testing, Documentation      | Complete project documentation, improve logging, perform integration testing, optimize performance, and prepare the final release. |

---

# 🏁 Expected Outcome

Upon completion of the six-week roadmap, the project will include:

* ✅ A complete console-based Cinema Booking System
* ✅ Layered architecture with clear separation of responsibilities
* ✅ Generic Repository implementation
* ✅ CSV-based data persistence
* ✅ Robust validation and exception handling
* ✅ Thread-safe concurrent ticket booking
* ✅ Application of multiple design patterns
* ✅ Unit and integration test coverage
* ✅ User, Developer, and Architecture documentation
* ✅ A production-ready GitHub portfolio project
